### 1. FIRST CONTACT (0-30 seconds)
1. Use [First Responder Checklist](./first-responder-checklist.md) to gather initial info
2. Determine severity: Emergency / Urgent / Standard / Low
3. Route accordingly:
   - ðŸ”´ **Emergency** â†’ [Emergency Procedures](#emergency-situations-immediate-response)
   - ðŸŸ¡ **Urgent** â†’ Escalate to Level 2 Support
   - ðŸŸ¢ **Standard** â†’ Continue with troubleshooting
   - âšª **Low** â†’ KB articles or schedule callback

### 2. DIAGNOSTIC PHASE (30 seconds - 2 minutes)
1. Check [Symptom Quick Lookup](#quick-lookup-by-symptom) above
2. If error code provided, go to appropriate [Error Code Reference](#error-code-references-by-system)
3. If vague description, use [Symptom-Based Troubleshooting Guide](./symptom-based-troubleshooting.md)
4. Verify system status with customer using [System Status Questions](#)

### 3. TROUBLESHOOTING PHASE (2-15 minutes)
1. Follow relevant KB article decision tree
2. Use technical manual for detailed procedures
3. Check [System Interaction Matrix](./system-interaction-matrix.md) if multiple systems affected
4. Document steps taken in ticket

### 4. RESOLUTION OR ESCALATION
1. If resolved, confirm with customer and close ticket
2. If not resolved within 15 minutes, check [Escalation Guide](./escalation-triage-guide.md)
3. Use [Customer Response Templates](./customer-response-templates.md) for communication
4. Hand off to Level 2 with complete diagnostic notes

## ðŸ’¡ PRO TIPS FOR SUPPORT AGENTS

### Efficiency Tips
1. **Bookmark this page** - It's your home base
2. **Learn the error code prefixes** - PWR, NAV, COM, SEN tell you the system immediately
3. **Use Ctrl+F on error code pages** - Fastest way to find specific codes
4. **Master the top 20 issues** - They cover 80% of calls
5. **Keep FTL checklist handy** - Second most common issue

### Customer Service Tips
1. **Stay calm for emergencies** - Your tone sets theirs
2. **Repeat critical safety info** - Don't assume they heard it the first time
3. **Explain the "why"** - Customers feel better when they understand
4. **Set expectations** - Tell them how long troubleshooting might take
5. **Document everything** - The next agent will thank you

### Technical Tips
1. **Power issues affect everything** - Check power first if multiple systems down
2. **Sensor errors cascade** - Bad sensor data causes false alarms elsewhere
3. **Coolant problems escalate fast** - Never dismiss coolant warnings
4. **FTL errors may be intentional safeties** - The system prevents dangerous jumps
5. **Environmental trends matter** - Slow decline vs. sudden drop changes diagnosis
